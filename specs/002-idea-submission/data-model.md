## Entities

### Idea

Primary entity representing a submitted innovation idea. Immutable after creation (no edit/delete in v1).

| Column | SQL Type | Constraints | Notes |
|--------|----------|-------------|-------|
| `id` | TEXT | PRIMARY KEY | UUID4, generated by app |
| `title` | TEXT | NOT NULL | ≤ 150 chars; enforced by CHECK + app-level Pydantic |
| `description` | TEXT | NOT NULL | ≤ 3,000 chars; enforced by CHECK + Pydantic |
| `category` | TEXT | NOT NULL | CHECK IN enum values below |
| `submitter_id` | TEXT | NOT NULL | FK → `users.id`; set from session, never from client |
| `submitted_at` | TEXT | NOT NULL, DEFAULT | ISO-8601 UTC via `strftime('%Y-%m-%dT%H:%M:%SZ', 'now')` |
| `attachment_filename` | TEXT | NULLABLE | Original filename as uploaded by the user |
| `attachment_stored_name` | TEXT | NULLABLE | `{uuid4}{ext}` — collision-safe name on disk |
| `attachment_mime_type` | TEXT | NULLABLE | Validated MIME type string |
| `attachment_size` | INTEGER | NULLABLE | File size in bytes |

**Attachment invariant**: All four `attachment_*` columns are either all NULL (no file) or all
non-NULL (file attached). Enforced by a CHECK constraint.

### Relationship to User

- `ideas.submitter_id` → `users.id` (many-to-one; one user, many ideas)
- No ORM `relationship()` or lazy-load: the submitter name is fetched via a single joined query
  on the list endpoint, not an N+1 loop.
- No cascade delete: ideas persist independently of user state (out of scope for v1).

## Category Enum

Categories are hardcoded in v1 (FR-015). Admin-managed categories are explicitly out of scope.

| DB value | Display label |
|----------|---------------|
| `process_improvement` | Process Improvement |
| `technology` | Technology |
| `cost_saving` | Cost Saving |
| `other` | Other |

Backend: enforced via `CheckConstraint` and `Literal` type in Pydantic schema.
Frontend: hardcoded `<SelectItem>` values in the shadcn/ui Select component.

## File Storage Layout

```
backend/
└── uploads/
    └── {idea_id}/
        └── {uuid4}{ext}     ← attachment_stored_name
```

- `idea_id/` subdirectory isolates each idea's file (one file max per idea in v1).
- `{uuid4}{ext}` prevents filename collision and path traversal attacks.
- The original filename (`attachment_filename`) is stored in the DB and sent as the
  `Content-Disposition: attachment; filename="{original}"` header on download.
- The `uploads/` root path is controlled by `settings.UPLOAD_DIR` (default: `./uploads`
  relative to the backend working directory). Added to `.gitignore`.

## Indices

| Index name | Column(s) | Direction | Purpose |
|------------|-----------|-----------|---------|
| `idx_ideas_submitted_at` | `submitted_at` | DESC | Default sort for list endpoint; avoids full table scan |
| `idx_ideas_submitter_id` | `submitter_id` | — | Efficient join/filter for submitter lookup |

SC-002 targets ≤ 1 s list load for ≤ 500 ideas. With `idx_ideas_submitted_at`, a `LIMIT 20 OFFSET 0`
query executes in single-digit milliseconds on SQLite.

## SQLAlchemy Model

```python
# backend/app/models/idea.py
from sqlalchemy import Column, String, Integer, Index, CheckConstraint
from app.models.user import Base


class Idea(Base):
    __tablename__ = "ideas"

    id = Column(String, primary_key=True)
    title = Column(String, nullable=False)
    description = Column(String, nullable=False)
    category = Column(String, nullable=False)
    submitter_id = Column(String, nullable=False)
    submitted_at = Column(
        String,
        nullable=False,
        server_default="(strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))",
    )
    attachment_filename = Column(String, nullable=True)
    attachment_stored_name = Column(String, nullable=True)
    attachment_mime_type = Column(String, nullable=True)
    attachment_size = Column(Integer, nullable=True)

    __table_args__ = (
        CheckConstraint(
            "category IN ('process_improvement','technology','cost_saving','other')",
            name="ck_ideas_category",
        ),
        CheckConstraint("length(title) <= 150", name="ck_ideas_title_length"),
        CheckConstraint("length(description) <= 3000", name="ck_ideas_description_length"),
        CheckConstraint(
            "(attachment_filename IS NULL) = (attachment_stored_name IS NULL)",
            name="ck_ideas_attachment_consistency",
        ),
        Index("idx_ideas_submitted_at", "submitted_at"),
        Index("idx_ideas_submitter_id", "submitter_id"),
    )
```

## Migration

`Base.metadata.create_all` in `init_db()` (called at startup) handles table creation.
`backend/app/database.py` must import `app.models.idea` alongside the existing Session import
so SQLAlchemy registers the `ideas` table before `create_all` runs.

```python
# backend/app/database.py additions
import app.models.idea  # noqa: F401 — ensure Idea model is registered
```

## Query Patterns

### Create idea (with optional file)
```sql
INSERT INTO ideas (id, title, description, category, submitter_id,
                   attachment_filename, attachment_stored_name, attachment_mime_type, attachment_size)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?);
```

### List ideas (newest first, paginated)
```sql
SELECT i.id, i.title, i.category, i.submitted_at,
       u.full_name AS submitter_name,
       (i.attachment_filename IS NOT NULL) AS has_attachment
FROM ideas i
JOIN users u ON u.id = i.submitter_id
ORDER BY i.submitted_at DESC
LIMIT ? OFFSET ?;
```
Uses `idx_ideas_submitted_at` for the ORDER BY; the JOIN uses `idx_users_email` (existing) or
a PK lookup on `users.id`.

### Get idea detail
```sql
SELECT i.*, u.full_name AS submitter_name
FROM ideas i
JOIN users u ON u.id = i.submitter_id
WHERE i.id = ?;
```

### Download attachment (after auth check in app layer)
```sql
SELECT attachment_stored_name, attachment_filename, attachment_mime_type, submitter_id
FROM ideas
WHERE id = ?;
```
